From Wordpress to Octopress

Nov 16th, 2013

For a long time I wasn’t happy with my Wordpress blog. Maintenance and hosting seemed unnecessary complex. When researching alternative solutions I found Octopress. In this article I highlight the reasons that made me switch.

Problems with Wordpress

Wordpress is a dynamic website with a database. As the code is open source it makes it easy for someone to write and use exploits to compromise your blog. That’s one reason why Wordpress and the plugins provide frequently updates. But even though I updated often I got a lot of spam comments. Even the captcha system I had in place didn’t stop them.

  • Changes to blog locally, then updating via FTP
  • Manual backups necessary (database, images, …)

Benefits of Octopress

  • Posts written in Markdown

  • Nice codeblock support

  • Static page generator

  • Website renders super fast as everything is static

  • Clear and modern HTML5, scss for Stylesheets

  • I use a modified version of the whitespace theme

  • Git provides with version history

  • 1 step deploy (rake deploy)

  • Hosting on GitHub (github.io)

  • Comment system is provided by external service

  • Updating blog via command line

Conclusion

One drawback is that you rely on external services. Imagine for some reason the comment provider stops his service. All your comments will be lost. I for myself rather take this risk in exchange for easier maintainance. I’m pretty happy with Octopress. If you are interested, go and check it out for yourself.
